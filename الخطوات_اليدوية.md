# دليل الخطوات اليدوية — منصة الأوس

هذا الملف يشرح **كل خطوة يدوية** يجب أن تنفّذها بنفسك حتى يعمل المشروع بشكل كامل وسحابي.
كل التعديلات البرمجية تمّت بالفعل على فرع `main`. تبقّى عليك الخطوات التالية فقط.

---

## ✅ أولاً: في الطرفية (Terminal) داخل VS Code

بعد فتح المشروع في VS Code، شغّل الأوامر التالية بالترتيب:

```bash
# 1. اسحب آخر التعديلات من GitLab (مهم جداً)
git pull origin main

# 2. ثبّت الحزم (في حال ظهرت أخطاء استيراد)
npm install

# 3. شغّل المشروع محلياً للتجربة
npm run dev
```

> لو ظهر خطأ في `git pull` بسبب تعارض مع تعديلات محلية عندك، شغّل:
> ```bash
> git stash        # يحفظ تعديلاتك المحلية مؤقتاً
> git pull origin main
> git stash pop    # يرجّع تعديلاتك (لو محتاجها)
> ```

---

## ✅ ثانياً: على لوحة تحكم Supabase (الأهم)

هذه الخطوة **ضرورية** — بدونها لن يعمل عزل الطلاب ولا الصفحات الجديدة.

1. افتح [supabase.com](https://supabase.com) وادخل على مشروعك.
2. من القائمة الجانبية اختر **SQL Editor**.
3. اضغط **New query**.
4. افتح ملف `supabase/rls_and_fixes.sql` من المشروع، **انسخ محتواه كاملاً**.
5. الصقه في محرر SQL واضغط **Run**.

> هذا الملف آمن للتشغيل أكثر من مرة (idempotent)، ولن يكرّر أي شيء.

هذا الملف يقوم بـ:
- إنشاء الجداول الناقصة: `lesson_progress`, `lesson_notes`, `discount_codes`, `platform_settings`.
- إضافة أعمدة جديدة لجدول `courses` (currency, features, tags, ... إلخ).
- تفعيل **عزل كامل لكل طالب** (RLS) على كل الجداول.
- إعطاء الأدمن صلاحية رؤية كل البيانات.

---

## ✅ ثالثاً: متغيرات البيئة (.env)

تأكد أن ملف `.env` (أو `.env.local`) موجود في جذر المشروع ويحتوي على:

```
NEXT_PUBLIC_SUPABASE_URL=رابط_مشروعك_على_سوبابيز
NEXT_PUBLIC_SUPABASE_ANON_KEY=المفتاح_العام_anon
SUPABASE_SERVICE_ROLE_KEY=مفتاح_السيرفس_رول
```

> هذه القيم تجدها في Supabase ← Settings ← API.
> **لا ترفع هذا الملف على GitLab أبداً** (موجود في .gitignore).

---

## ✅ رابعاً: إنشاء حساب الأدمن

العزل الجديد يميّز الأدمن عن الطالب عبر حقل `role = 'admin'` في جدول `profiles`.
لجعل حسابك أدمن:

1. سجّل حساباً عادياً من صفحة التسجيل في الموقع.
2. ادخل على Supabase ← **Table Editor** ← جدول `profiles`.
3. ابحث عن صفّك (بالإيميل أو الجوال).
4. غيّر قيمة العمود `role` إلى `admin` واحفظ.

بعدها سجّل دخول من جديد، وستدخل تلقائياً على بوابة الإدارة.

---

## ⚠️ خامساً: أمور ناقصة تحتاج خدمات خارجية (اختيارية لاحقاً)

هذه ليست بيانات وهمية، لكنها مزايا تحتاج إعداد خدمة خارجية لتعمل فعلياً:

1. **رفع الفيديو/الملفات مباشرة من الجهاز** (في صفحات الدروس/المكتبة)
   - حالياً: الرفع المباشر يحفظ اسم الملف فقط (غير فعلي).
   - يعمل الآن بشكل كامل عبر خيار **"رابط خارجي"**.
   - للرفع الحقيقي: يجب تفعيل **Supabase Storage** وربطه (خطوة لاحقة).

2. **إشعارات واتساب لأولياء الأمور** (`/admin/whatsapp`)
   - حالياً: واجهة جاهزة، لكن الإرسال محاكاة.
   - للتفعيل الحقيقي: تحتاج اشتراك في **WhatsApp Business API** (مثل Twilio أو Meta) ومفتاح API.
